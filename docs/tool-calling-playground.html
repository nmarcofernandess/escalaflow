As 5 Famílias

1. consultar_contexto READ

Consulta detalhada quando o contexto automático não basta.

  • entidade: 'setor' → consultar
  • entidade: 'colaborador', id: 5 → buscar_colaborador
  • entidade: 'escala' → consultar
  • entidade: 'regras' → consultar

2. editar_ficha WRITE

Cria, atualiza ou remove qualquer entidade.

  • posto, criar → salvar_posto_setor
  • regra_horario → salvar_regra_horario_colaborador
  • perfil_horario, remover → deletar_perfil_horario
  • colaborador, id:5 → atualizar
  • excecao, criar → criar
  • demanda + data_especifica → salvar_demanda_excecao_data

3. executar_acao ACTION

Comandos de domínio com side effects.

  • gerar_escala → gerar_escala
  • oficializar → oficializar_escala
  • ajustar_celula → ajustar_alocacao
  • preflight → preflight
  • diagnosticar → diagnosticar_escala
  • backup → fazer_backup

4-5. Memórias MEMORY

Passthrough direto — mesma interface de antes.

  • salvar_memoria → salvar_memoria
  • remover_memoria → remover_memoria
