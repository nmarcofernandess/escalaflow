O Princípio: Contexto é Leitura, Tool é Escrita

O que o CONTEXTO faz (automático, pre-LLM)

Injetado no system prompt ANTES da IA pensar. Ela já sabe tudo isso quando lê a pergunta.

  • AUTO Memórias do RH (todas, sempre)
  • AUTO Knowledge RAG (busca semântica na pergunta)
  • AUTO Preview de ciclo (cobertura, déficit, folgas)
  • AUTO Equipe completa (colaboradores, postos, regras)
  • AUTO Demanda planejada (por dia e faixa)
  • AUTO Escala atual (status, violações)
  • AUTO Contratos + perfis de horário
  • AUTO Alertas ativos
  • AUTO Feriados próximos
  • AUTO Regras com override

O que as TOOLS fazem (sob demanda, pós-LLM)

Só quando o contexto não basta ou precisa ESCREVER algo.

  • TOOL Consulta detalhada (filtros específicos)
  • TOOL Busca explícita no RAG (se auto-RAG falhou)
  • TOOL Criar/atualizar/remover registros
  • TOOL Salvar/apagar memórias
  • TOOL Gerar escala (motor OR-Tools)
  • TOOL Oficializar escala
  • TOOL Preflight, diagnóstico
  • TOOL Backup

As 3 Famílias (Fase 2)

1. consultar_contexto READ

Leitura sob demanda. Só quando o contexto automático não basta.

  • setor → consultar
  • colaborador + id → buscar_colaborador
  • escala, regras, contrato → consultar
  • conhecimento → buscar_conhecimento

★ NOVO: busca RAG explícita quando auto-RAG falhou

2. editar_ficha WRITE

Cria, atualiza ou remove qualquer entidade.

  • colaborador, setor, excecao... → criar/atualizar/deletar
  • posto → salvar_posto_setor
  • regra_horario → salvar_regra_horario
  • memoria, criar → salvar_memoria
  • memoria, remover → remover_memoria

★ NOVO: memória absorvida como entidade

3. executar_acao ACTION

Comandos de domínio com side effects.

  • gerar_escala → gerar_escala
  • oficializar → oficializar_escala
  • ajustar_celula → ajustar_alocacao
  • preflight → preflight
  • diagnosticar → diagnosticar_escala
  • backup → fazer_backup

★ = seção que garante que a IA não faz cagada naquele assunto

AUTO ★ Memórias do RH — sempre presentes, toda mensagem ▼ 
### Memórias do RH (3/50)
- Cleunice nunca troca turno com ninguém
- Black Friday precisa de 8 pessoas no Caixa
- Maria Clara só pode trabalhar DOM e SEG (acordo verbal)

★ A IA vê TODAS as memórias em TODA mensagem. Se alguém pergunta "a Cleunice pode trocar turno?", a IA já sabe: NÃO. Zero tool calls.

Pra SALVAR nova memória → editar_ficha({ entidade: 'memoria', operacao: 'criar', dados: { conteudo: '...' } })

Pra APAGAR → editar_ficha({ entidade: 'memoria', id: 3, operacao: 'remover' })

AUTO ★ Knowledge RAG — busca automática por embedding na pergunta ▼ 
### Conhecimento relevante (confiança: 78%)
- sistema — regras-colaborador: Regras de horário, folga fixa, folga variável, tipo A/B
- sistema — perguntas-frequentes: FAQ sobre ciclo, cobertura, oficialização
- sistema — fluxos-trabalho: Passo a passo para gerar e oficializar escalas

★ Roda AUTOMATICAMENTE se a pergunta tem mais de 10 caracteres. Busca semântica (embeddings) nos 50 chunks da knowledge base. A IA já recebe os docs relevantes ANTES de pensar.

Se score baixo e a IA precisa de mais → consultar_contexto({ entidade: 'conhecimento', filtros: { consulta: 'query reformulada' } })

Retry natural: o Vercel AI SDK roda até 10 steps de tool calls. Se o RAG volta fraco, a IA reformula e tenta de novo.

Comparações: Antes vs Agora

Exemplo: "lembra que a Cleunice não pode trabalhar sábado"

ANTES (30 tools):

1. salvar_memoria({ conteudo: "Cleunice não pode sábado" })
1 tool call — tool dedicada entre 30 opções
→

FASE 2 (3 tools):

1. editar_ficha({ entidade: "memoria", operacao: "criar", dados: { conteudo: "Cleunice não pode sábado" } })
↳ adapter roteia → salvar_memoria
1 tool call — mesma família que editar colaborador

Exemplo: "o que diz a CLT sobre hora extra?"

ANTES (30 tools):

1. buscar_conhecimento({ consulta: "CLT hora extra" })
1 tool call — tool dedicada. Sem retry automático.
→

FASE 2 (3 tools):

0 tool calls — auto-RAG já injetou no contexto!
discovery._autoRag("CLT hora extra") → encontrou docs relevantes
Se score baixo → IA chama:
consultar_contexto({ entidade: "conhecimento", filtros: { consulta: "CLT Art 59 hora extra limite" } })
↳ adapter roteia → buscar_conhecimento
↳ score ainda baixo? → IA reformula e tenta de novo (até 3x no tool loop)

Exemplo: "a distribuição de folgas da padaria tá boa?"

ANTES (30 tools, sem preview):

1. consultar({ entidade: "setores", filtros: { nome: "Padaria" } })
2. consultar({ entidade: "colaboradores", filtros: { setor_id: 4 } })
3. consultar({ entidade: "alocacoes", filtros: { setor_id: 4 } })
3+ tool calls. IA cega pro preview.
→

FASE 2 (3 tools, com preview no contexto):

0 tool calls — responde direto do contexto!
Preview: Ciclo 2 semanas, Cobertura 100%, Déficit 0
⚠️ Conflito folga variável em SEG: 3/4
+ Memórias: "Maria Clara só pode DOM e SEG"
A IA cruza preview + memórias + equipe pra responder.

Exemplo: "muda o horário da Cleunice pra entrar às 09:00"

ANTES (30 tools):

1. buscar_colaborador({ nome: "Cleunice" })
2. salvar_regra_horario_colaborador({ colaborador_id: 1, hora_inicio_min: "09:00" })
2 tool calls. Precisou buscar antes.
→

FASE 2 (3 tools):

1. editar_ficha({ entidade: "regra_horario", dados: { colaborador_id: 1, hora_inicio_min: "09:00" } })
↳ Cleunice ID:1 já estava no contexto (equipe)
↳ adapter roteia → salvar_regra_horario_colaborador
1 tool call. ID veio do contexto.
