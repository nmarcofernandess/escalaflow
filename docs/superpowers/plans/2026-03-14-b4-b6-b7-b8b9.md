### Task 3: Implement demand-aware folga distribution

**Files:**
- Modify: `src/shared/simula-ciclo.ts:267-282` (Step 2 of gerarCicloFase1)

The key insight: instead of `base1 = forcada?.folga_fixa_dia ?? (p % 6)`, compute the best day based on surplus = N - demand[day].

- [ ] **Step 1: Replace Step 2 folga logic**

In `gerarCicloFase1`, replace the Step 2 block (~lines 267-295) with:

```typescript
  // --- Step 2: Folgas semanais 5x2 (2 por semana) ---
  const hasDemanda = input.demanda_por_dia && input.demanda_por_dia.length >= 6

  // Track folgas assigned per weekday across all people (for demand-aware spreading)
  // folgaCount[d] increments each time a person gets folga on day d
  const folgaCount = [0, 0, 0, 0, 0, 0] // SEG-SAB

  // Helper: pick best folga day considering demand surplus AND already-assigned folgas
  // effective_surplus = (N - demanda[d]) - folgaCount[d]
  // Higher effective_surplus = less impact of adding another folga on that day
  const pickBestFolgaDay = (demanda: number[], exclude: number | null): number => {
    let bestDay = 0
    let bestScore = -Infinity
    for (let d = 0; d < 6; d++) { // SEG-SAB only (0-5)
      if (d === exclude) continue
      const surplus = N - (demanda[d] ?? 0)
      const score = surplus - folgaCount[d] // factor in already-assigned folgas
      if (score > bestScore) {
        bestScore = score
        bestDay = d
      }
    }
    return bestDay
  }

  for (let p = 0; p < N; p++) {
    const forcada = input.folgas_forcadas?.[p]
    const isFixaDom = forcada?.folga_fixa_dom === true

    // base1: forced > demand-aware > round-robin fallback
    const base1 = forcada?.folga_fixa_dia
      ?? (hasDemanda ? pickBestFolgaDay(input.demanda_por_dia!, null) : (p % 6))
    folgaCount[base1]++

    // base2: forced > demand-aware (exclude base1) > round-robin fallback
    const base2 = isFixaDom ? null
      : (forcada?.folga_variavel_dia
        ?? (hasDemanda ? pickBestFolgaDay(input.demanda_por_dia!, base1) : ((p + 3) % 6)))
    if (base2 != null) folgaCount[base2]++

    for (let w = 0; w < weeks; w++) {
      const sundayOff = grid[p][w * 7 + 6] === 'F'
      if (sundayOff) {
        grid[p][w * 7 + base1] = 'F'
      } else {
        grid[p][w * 7 + base1] = 'F'
        if (base2 != null) grid[p][w * 7 + base2] = 'F'
      }
    }
  }
```

**How it works:**
- Maintains `folgaCount[d]` — tracks how many people already have folga on each day
- `pickBestFolgaDay` calculates `score = (N - demanda[d]) - folgaCount[d]` — higher score = more room for another folga
- This naturally SPREADS folgas across days: as more people get folga on day X, its score drops
- If `demanda_por_dia` not provided: falls back to old `p % 6` behavior (backward compatible)
- Forced folgas always win (override demand-aware logic) and increment folgaCount
- `folga_fixa_dom` logic from B3 is preserved

```typescript
/**
 * B7: Derive tipo_trabalhador from contract name.
 * Ensures consistency even if the DB column is stale.
 * Mirrors frontend derivarTipoTrabalhadorPorContrato() in ColaboradorDetalhe.tsx.
 */
function derivarTipoTrabalhador(
  tipoColuna: string | null,
  contratoNome: string,
): string {
  // NFD normalization to strip accents (matches frontend logic)
  const nome = contratoNome.normalize('NFD').replace(/[\u0300-\u036f]/g, '').toLowerCase()
  if (nome.includes('intermit')) return 'INTERMITENTE'
  if (nome.includes('estagi')) return 'ESTAGIARIO'
  return tipoColuna || 'CLT'
}
```

This mirrors the frontend's `derivarTipoTrabalhadorPorContrato()` in ColaboradorDetalhe.tsx but runs in the backend as a guard.
